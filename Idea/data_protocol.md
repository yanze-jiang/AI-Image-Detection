# 数据与实验协议

## 1. 主数据集选择

本项目正式实验以 `CIFAKE` 为主，不再使用 `GenImage`，原因如下：

1. `GenImage` 体量过大，不适合当前硬件条件和存储条件。
2. `CIFAKE` 总规模约 `120,000` 张图像，下载体量约 `100MB` 级别，更适合作为课程项目的主实验数据。
3. `CIFAKE` 自带清晰的训练集和测试集划分，便于快速搭建 baseline 和改进模型。

## 2. 数据集结构

`CIFAKE` 的标准目录结构通常如下：

1. `train/REAL`
2. `train/FAKE`
3. `test/REAL`
4. `test/FAKE`

其中：

1. `REAL` 图像来自 `CIFAR-10`
2. `FAKE` 图像由 `Stable Diffusion v1.4` 生成
3. 图像尺寸为 `32 x 32`

## 3. 推荐实验规模

为了兼顾训练速度和结果稳定性，建议采用下面两级配置：

1. **标准配置**
   - 训练集：使用 `train` 全量
   - 测试集：使用 `test` 全量

2. **轻量配置**
   - 训练集：从 `train/REAL` 和 `train/FAKE` 中各抽 `10,000` 张
   - 验证集：从训练集中再划分 `2,000 + 2,000`
   - 测试集：使用 `test` 全量

如果算力更紧张，也可以把训练样本进一步减到每类 `5,000` 张，但测试集尽量保留完整，以保证结果更稳定。

## 4. 预处理与偏差控制

所有图像统一采用相同预处理流程：

1. 将原始 `32 x 32` 图像上采样到 `256 x 256`
2. 模型输入阶段中心裁剪到 `224 x 224`
3. 所有图片统一保存为 `JPEG`，质量系数固定为 `95`
4. 去除明显损坏或无法读取的样本
5. 保持真假样本数量平衡

这样做的目的，是避免模型主要依赖：

1. PNG 与 JPEG 的格式差异
2. 原始分辨率差异
3. 明显的数据清洗痕迹

## 5. 任务定义

本项目固定做 3 个任务，不再扩展到更多变体：

1. `In-domain binary classification`  
   在 `CIFAKE` 标准训练集上训练，并在标准测试集上评估，验证模型是否具备基本判别能力。

2. `Robustness under post-processing`  
   对测试图像施加 JPEG 压缩和缩放，再评估性能下降情况。

3. `Data-efficiency comparison`  
   比较轻量训练集和标准训练集下的结果差异，说明方法是否适合资源受限环境。

## 6. 后处理鲁棒性设置

建议至少做以下两类扰动：

1. JPEG 压缩：质量系数 `95 / 85 / 75`
2. 缩放扰动：先缩小到 `128 x 128`，再恢复到 `224 x 224`

如果时间不足，优先保留 JPEG 压缩实验，因为它更符合图像传播中的常见场景。

## 7. 评价指标

最终报告至少统一汇报以下指标：

1. `Accuracy`
2. `AUC`
3. `F1-score`

其中 `AUC` 作为主指标，`Accuracy` 和 `F1-score` 作为辅助指标。因为在不同生成器和不同扰动条件下，AUC 更适合横向比较。

## 8. 结果记录模板

每次实验至少记录以下字段：

1. 数据子集版本
2. 是否做偏差控制
3. 模型名称
4. 训练样本规模
5. 测试设置
6. 是否加压缩/缩放扰动
7. Accuracy / AUC / F1
8. 备注

## 9. 本文档对应的执行结论

当前推荐的正式实验口径为：

1. 主数据集：`CIFAKE`
2. 主任务：真假分类 + 鲁棒性测试 + 轻量训练规模对比
3. 主指标：`AUC`
4. 主原则：先保证实验协议轻量可运行，再比较模型效果
